=head1 NAME

B<Table Numbers>, a collection of javascript assignments for use with
the B<Table Maker> C<div_table()> (or C<table_maker()>) function (see
L<http://dev.galileo.usg.edu/table_maker_docs.html>).

=head1 SYNOPSIS

 <html>
 <head>
 <script type="text/javascript" src="/javascript/table_maker/table_numbers.js"></script>
 <script type="text/javascript" src="/javascript/table_maker/table_maker.js"></script>
 </head>
 <body onload="table_maker(hflmrmff)"> <!-- HeaderFull LeftMiddle RightMiddle FooterFull -->
 <div id="page">
     <div id="header"       > Header       </div>
     <div id="left_margin"  > Left Margin  </div>
     <div id="body"         > Body         </div>
     <div id="right_margin" > Right Margin </div>
     <div id="footer"       > Footer       </div>
 </div>
 </body>
 </html>

The above example illustrates including the files C<table_numbers.js>
and C<table_makers.js>, and it illustrates calling the C<table_maker()>
function with the predefined variable C<hflmrmff> (defined in
table_numbers.js).

=head1 DESCRIPTION

B<Table Numbers> is collection of predefined javascript assignments.
These assign table specs objects to the variables named C<t0> - C<t46>
(and C<c3>), as well as to these variables' aliases, e.g., C<bf>
(B<C<b>>ody B<C<f>>ull), C<hflbrbfc> (B<C<h>>eader B<C<f>>ull, B<C<l>>eft B<C<b>>ottom,
B<C<r>>ight B<C<b>>ottom, B<C<f>>ooter B<C<c>>enter), etc.  The variables names and
their alias are itemized below.

The intention behind this is to give the programmer clear examples of
the most common Web page layout schemes, i.e., layouts that have a
header, a left margin, possibly a right margin, the main content of the
page (here referred to as the C<body>, and a footer.  The assignments
C<t0> - C<t46> offer 47 permutations of how those five elements of the
layout might be arranged.  In addition, the variable C<c3> is included
as an example of how one might lay out a page with a header and three
columns.

There are intentional assumptions made in these assignments, e.g., the
header div has the id C<"header">, the div that will become the page
layout table has the id C<"page">, etc.  As a page designer, you may
choose to follow those assumptions (as illustrated above) or not.  If
you do, you can call C<div_table()> or C<table_maker()> with the
variable names defined in C<table_numbers.js>, e.g. C<div_table(t10)>.
If instead, you want to name your divs differently, then you can use
the assignments in C<table_numbers.js> as examples for creating your
own table specs assignments.

=head2 A note about aliases

Since the variable names C<"t0"> - C<"t46"> are not very descriptive,
aliases have been included that follow this naming scheme:

  h header
  l left margin
  r right margin
  f footer
 (b body) (body is implicit, except in the 'bf' alias)

  f  full   (all)                       (hf lf rf ff bf)
  l  left   (header, footer)            (hl fl)
  r  right  (header, footer)            (hr fr)
  c  center (header, footer)            (hc fc)
  t  top    (left margin, right margin) (lt rt)
  b  bottom (left margin, right margin) (lb rb)
  m  middle (left margin, right margin) (lm rm)

For example, table 13 (C<t13>) has the alias C<hllbrffc>, which stands
for B<C<h>>eader B<C<l>>eft, B<C<l>>eft margin B<C<b>>ottom, B<C<r>>ight margin
B<C<f>>ull, B<C<f>>ooter B<C<c>>enter Note that B<C<b>>ody is assumed to be present
in all tables, so it is not included in any aliases except for
table 0 (C<t0>) that has I<only> a body: C<bf>.

Another example, table 31 (C<t31>) (aka the "gun"), has the alias
C<hflb>, B<C<h>>eader B<C<f>>ull, B<C<l>>eft margin B<C<b>>ottom.  This illustrates
that a table layout might not include all of the elements: header, left
margin, right margin, and footer.

However, the assignments assume that the page has all of those divs.
The divs that do not have a corresponding table layout element are
moved into the same table cell as the body div in the order indicated
in the assignment.  So since table 31 does not include table cells for
right margin and footer, the right margin and footer divs are moved
into the body cell (C<id="td_body">) along with the body div.
